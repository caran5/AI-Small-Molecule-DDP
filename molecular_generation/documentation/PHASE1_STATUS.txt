## What Was Built

### 1️⃣ Conditional Generation (Property Steering)

Enables chemists to **control** molecular generation by specifying target properties.

**Key Files**:
- `src/models/unet.py` - **ConditionalUNet** class (+100 lines)
- `src/data/preprocessing.py` - **PropertyNormalizer** class (+110 lines)
- `src/data/loader.py` - **ConditionalMoleculeDataLoader** class (+100 lines)
- `src/inference/generate.py` - **generate_with_properties()** & **ConditionalGenerationPipeline** (250 lines)
- `scripts/train_conditional.py` - **ConditionalTrainer** for training (300 lines)

**Usage Example**:
```python
from src.models.unet import ConditionalUNet
from src.inference.generate import generate_with_properties

model = ConditionalUNet(input_dim=100, n_properties=5)
target = {'logp': 3.5, 'mw': 400, 'hbd': 2, 'hba': 5, 'rotatable': 4}
samples = generate_with_properties(model, target, num_samples=50)
```

### 2️⃣ Validation Metrics Beyond Loss

Comprehensive metrics to evaluate generation quality (not just loss).

**Key File**: `src/eval/metrics.py` (450 lines)

**Implemented Metrics**:
- **chemical_validity()** - % valid SMILES (target: >85%)
- **diversity_metric()** - Pairwise variation (target: 0.4–0.6)
- **property_fidelity()** - Error vs target properties (target: <0.1 MSE)
- **distribution_distance()** - MMD/Wasserstein distance (target: 0.1–0.3)
- **novel_statistics()** - Out-of-distribution fraction (target: 10–30%)
- **compute_all_metrics()** - Unified interface

**Usage Example**:
```python
from src.eval.metrics import compute_all_metrics, print_metrics

metrics = compute_all_metrics(
    generated_smiles,
    generated_features,
    training_features,
    target_properties
)
print_metrics(metrics, epoch=1)
```

## Complete Workflow

```
1. Prepare Data
   ├─ Encode molecules to features [batch, 100]
   └─ Extract properties [batch, 5]
                    ↓
2. Create DataLoader
   ├─ ConditionalMoleculeDataLoader
   ├─ Auto-fits PropertyNormalizer
   └─ Returns batches: (features, properties)
                    ↓
3. Train Model
   ├─ ConditionalUNet with property conditioning
   ├─ Early stopping (patience=5)
   └─ Save best model
                    ↓
4. Train Ensemble (Optional)
   ├─ 3 independent models with different seeds
   ├─ Each trains with different initialization
   └─ Save checkpoints/ensemble/model_*.pt
                    ↓
5. Generate Candidates
   ├─ Ensemble generation with uncertainty
   ├─ Confidence filtering (remove std > threshold)
   ├─ Decode to SMILES
   ├─ Compute properties
   └─ Rank by fidelity
                    ↓
6. Evaluate & Select
   ├─ Check validity, diversity, fidelity
   ├─ Select top-N candidates
   └─ Ready for downstream evaluation
```

## Key Design Decisions

### Why PropertyNormalizer?
- Handles different scales (LogP: 0–5, MW: 100–1000, HBD: 0–10)
- Robust to distribution skewness
- Reusable across models

### Why Ensemble Over Bayesian?
- Faster inference (no posterior sampling)
- Simpler (train independent models)
- Parallelizable
- Direct uncertainty via model disagreement

### Why These 5 Properties?
**Lipinski's Rule of Five + 1**:
- LogP: membrane permeability
- MW: absorption
- HBD/HBA: solubility/selectivity
- Rotatable bonds: flexibility

### Why Multiple Metrics?
- Validity catches syntax errors
- Diversity catches mode collapse
- Fidelity catches conditioning failure
- MMD catches overfitting
- Novelty catches memorization
